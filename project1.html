Portfolio Website

A responsive, animated personal portfolio built from scratch with HTML, CSS, and JavaScript.

HTML5 CSS3 JavaScript Figma GitHub Pages
⚡ Status: Live

Overview

No frameworks, no templates, no page builders. I wrote this site from scratch because I wanted to actually understand what I was building. Every animation, layout, and interaction is hand-coded.

Fully responsive - works properly on mobile, tablet, and desktop.
No Bootstrap, no Tailwind, no frameworks. Just HTML, CSS, and vanilla JS.
Dark theme with a red accent - designed it myself, not picked from a template.
Scroll-triggered animations built with IntersectionObserver - no libraries.

Design & Planning

I did a rough plan in Figma before touching any code. Wanted to know where everything was going before I started building it.

Wireframing

Simple low-fidelity wireframes in Figma mapped out the page structure - hero, about, qualifications, projects, and contact - before any visual design decisions were made.

Visual Direction

Off-black backgrounds (#111), a bold red accent (#ff4b2b), and Poppins as the typeface. Minimal use of colour to keep things sharp and focused on the content.

Structure

Planned as a single-page site with smooth anchor scrolling, a mobile burger menu, and separate HTML pages for individual project deep-dives.

Tool

Quick wireframes in Figma to map out the structure before writing any HTML.

Development

Everything written by hand. Separate CSS and JS files per page to keep things clean and easy to maintain.

HTML5

Semantic, accessible markup throughout. Each page is its own HTML file with clean structure and proper heading hierarchy.

● Core

CSS3

Modular CSS per page - styles.css for the homepage, about.css, projects.css, and page-specific overrides. Flexbox and CSS Grid for all layouts. Full mobile responsiveness with media queries.

● Core

JavaScript

Small focused JS files for each behaviour - burger menu, scroll animations, hero entrance, form handling, and project animations. No libraries.

● Core

Animations

IntersectionObserver API used throughout for scroll-triggered entrance animations. CSS transitions handle hover states and micro-interactions.

● Core

Contact Form

Form submissions handled via Formspree - no backend required. Includes client-side validation and a redirect to a thank you page on success.

● Live

Responsive

Fully responsive across all breakpoints. Burger menu on mobile, fluid grids, and scaled typography using clamp() for smooth scaling.

● Live
Build

Built in VS Code. Separate files per page so nothing becomes a mess to maintain.

Deployment

Hosted on GitHub Pages. Free, straightforward, and deploys automatically when I push to main.

GitHub Pages

Served straight from the GitHub repo. No server to manage, no cost, and it deploys automatically on push.

Version Control

Full Git history from the first commit. Every change is tracked - useful when something breaks and you need to go back.

Future Plans

Custom domain is on the list. Eventually want to add a blog and maybe a proper backend, but not until there's a good reason to.

Hosting

GitHub Pages handles HTTPS and deploys on every push to main. Does the job well.
